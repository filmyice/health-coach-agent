# 1. 시스템 목적

본 시스템은 단순 비타민 추천 앱이 아니라 다음 4가지를 하나의 흐름으로 제공하는 **경량 건강 코치 시스템**이다.

- 음식 추천
- 운동/생활습관 추천
- 비타민/영양 성분 추천
- 추천 영양 성분에 대한 쇼핑 가격 비교 및 판매처 안내

---

# 2. 배경 및 목적

많은 사용자가 피로, 수면 부족, 눈 피로, 식습관 불균형, 운동 부족 등의 이유로 건강 관리를 원하지만, 복잡한 설문이나 의학적 정보 입력을 부담스러워한다.

기존 건강 추천 서비스는 두 가지 문제가 있다. 입력 항목이 너무 많아 사용자가 중도 이탈하며, 영양제 위주 추천으로 인해 상업적 인상이 강하고 신뢰도가 낮아진다.

따라서 본 시스템은 **최소 입력**만으로 우선 간단한 건강 제안을 제공하고, 추천 결과도 **음식 → 생활습관 → 영양 성분** 순으로 제시하여 실천 가능성과 신뢰도를 높인다.

---

# 3. 시스템 범위

## 포함
- 최소 입력 기반 건강 목표 파악
- 1차 추천 생성 및 동적 보정 질문
- 음식 / 운동·생활습관 / 비타민·영양 성분 추천
- 영양 성분별 주의사항 표시 및 위험 플래그 확인
- 추천 성분 관련 쇼핑 상품 탐색 (Playwright 크롤링)
- 가격 비교 및 판매처 안내

## 제외
- 질병 진단, 의료행위 대체, 의약품 처방
- 실시간 의료기록 연동, 직접 결제·배송 처리
- 건강 상태의 임상적 확정 판단

# 5. 서비스 원칙 (전 영역 공통)

1. **입력 최소화 우선**: 정밀도보다 사용자 입력 피로 감소를 우선한다.
2. **단계적 정밀화**: 초기에는 최소 입력으로 1차 추천을 제공하고, 추가 질문은 필요한 경우에만 1~2개 노출한다.
3. **건강 코치형 결과 제공**: 영양 성분만이 아니라 음식과 생활습관까지 함께 제안한다.
4. **비의료적 표현 유지**: 진단, 처방, 치료로 오인될 표현을 사용하지 않는다.
5. **설명 가능성 확보**: 왜 이런 추천이 나왔는지 간단히 설명한다.
6. **고위험 상황 보수적 처리**: 임신·수유, 복용약, 기저질환, 알레르기 등은 경고 및 상담 권고를 우선한다.
7. **주의사항 분리 표기**: 영양 성분별 주의사항은 추천 이유와 분리해 별도 섹션으로 표시한다.
8. **크롤링 실패 격리**: 쇼핑 크롤링 실패 시 전체 흐름이 중단되지 않아야 한다.

---

# 6. 추천 우선순위 원칙 (전 영역 공통)

최종 결과는 반드시 다음 순서로 제시한다.

```
음식 → 운동/생활습관 → 영양 성분 → 성분별 주의사항 → 쇼핑 링크
```

---

# 7. 용어 정의

| 용어 | 정의 |
|------|------|
| 건강 목표 | 사용자가 개선하고 싶은 대표 건강 관심사 |
| 1차 추천 | 최소 입력만으로 생성한 초기 추천 |
| 보정 질문 | 추천 정확도 향상을 위해 필요한 경우에만 제시하는 추가 질문 |
| 위험 플래그 | 안전상 주의가 필요한 조건 |
| 음식 추천 | 사용자가 바로 실행할 수 있는 식품 수준의 제안 |
| 생활습관 추천 | 짧고 쉬운 행동 중심 제안 |
| 영양 성분 추천 | 비타민, 미네랄, 기능성 영양 성분 제안 |
| 성분별 주의사항 | 특정 영양 성분 복용 전 확인해야 할 금기, 중복 섭취, 상호작용, 유의점 |
| 쇼핑 연동 | 추천 성분에 맞는 상품 후보 탐색 및 가격 비교 |
| 스킬 | 기능 단위의 도구/스크립트 |
| 서브에이전트 | 특정 역할을 맡는 독립 지침 단위의 에이전트 |

# 8. 판단과 코드의 역할 분리 원칙

| 에이전트가 직접 수행 | 스크립트로 처리 |
|---------------------|----------------|
| 건강 목표 해석 | 입력 스키마 검증 |
| 보정 질문 선택 | 데이터 정규화 |
| 음식/습관/영양 추천 설명 생성 | 규칙 기반 추천 후보 계산 |
| 주의사항 요약 문구 생성 | 위험 플래그 계산 |
| 톤 조정, 정성적 검토, 자기 검증 | Playwright 크롤링, 가격 계산, 파일 저장 |
