# 1. 프론트엔드 역할

프론트엔드는 다음 두 가지 역할을 담당한다.

- 사용자 입력(Level 1~3)을 수집해 **JSON으로 에이전트에 전달**한다.
- 에이전트가 반환한 **최종 JSON/Markdown 결과를 사용자에게 렌더링**한다.

에이전트 내부 로직(추천, 크롤링, 검증)은 프론트엔드 관심사가 아니다.

---

# 2. 입력 흐름

## 2.1 전체 입력 단계

```
[Step 1] Level 1 입력 (필수, 최초 1회)
    → 에이전트에 JSON 전달
    → 에이전트가 보정 질문 필요 여부 판단

[Step 2] Level 2 보정 질문 (조건부, 최대 2개)
    ← 에이전트가 질문 목록 반환
    → 사용자 선택 → JSON으로 재전달

[Step 3] Level 3 안전 확인 질문 (조건부, 고위험 상황만)
    ← 에이전트가 안전 질문 반환
    → 사용자 선택 → JSON으로 재전달
```

## 2.2 Level 1: 최소 필수 입력

초기 화면에서 수집하는 3가지 항목이다. 모두 선택형 UI로 제공한다.

**건강 목표** (단일 선택, 필수)

| 선택지 |
|--------|
| 피로 관리 |
| 수면 관리 |
| 면역 관리 |
| 눈 건강 |
| 장 건강 |
| 뼈 건강 |
| 피부·항산화 |

**나이대** (단일 선택)

10대 / 20대 / 30대 / 40대 / 50대 이상 / 답변 안 함

**성별** (단일 선택)

여성 / 남성 / 답변 안 함

### 전달 JSON 포맷

```json
{
  "health_goal": "피로 관리",
  "age_group": "30대",
  "gender": "여성"
}
```

모든 항목에 "건너뛰기" 또는 "모르겠어요" 옵션을 제공하며, 이 경우 `"unknown"`으로 전달한다.

## 2.3 Level 2: 동적 보정 질문

에이전트가 반환한 질문 목록을 렌더링한다. 최대 2개, 선택형 UI만 사용한다.

에이전트가 반환하는 질문 예시:
- 실내 생활이 많은가요?
- 식사가 불규칙한가요?
- 잠이 부족한가요?
- 화면을 오래 보나요?
- 운동이 부족한 편인가요?

### 전달 JSON 포맷

```json
{
  "refinement_answers": [
    { "question": "실내 생활이 많은가요?", "answer": "예" },
    { "question": "잠이 부족한가요?", "answer": "예" }
  ]
}
```

## 2.4 Level 3: 안전 확인 질문

고위험 상황(위험 플래그 감지 시)에서만 에이전트가 요청한다.

에이전트가 반환하는 질문 예시:
- 복용 중인 약이 있나요?
- 임신·수유 중인가요?
- 알레르기나 기저질환이 있나요?
- 이미 복용 중인 영양제가 있나요?

### 전달 JSON 포맷

```json
{
  "safety_answers": [
    { "question": "복용 중인 약이 있나요?", "answer": "없음" },
    { "question": "임신·수유 중인가요?", "answer": "아니요" }
  ]
}
```

# 3. 출력 화면 설계

## 3.1 결과 표시 순서 (고정)

에이전트가 반환하는 `user_result.json`을 다음 순서로 렌더링한다.

```
1. 음식 추천
2. 운동/생활습관 추천
3. 영양 성분 추천
4. 성분별 주의사항
5. 쇼핑 정보 (보조 섹션)
```

## 3.2 각 섹션 렌더링 규칙

### 음식 추천 섹션
- 2~4개의 식품을 카드 또는 리스트로 표시한다.
- 각 항목에 간단한 추천 이유 1줄을 함께 표시한다.

### 운동/생활습관 추천 섹션
- 1~3개의 행동 중심 제안을 리스트로 표시한다.
- 항목당 1~2줄 이내의 짧은 설명을 제공한다.

### 영양 성분 추천 섹션
- 1~3개의 성분을 카드 형태로 표시한다.
- 각 성분 카드에는 **추천 이유**와 **주의사항** 섹션을 분리해 고정 배치한다.

### 주의사항 UI 규칙
- 성분 카드마다 주의사항 섹션을 반드시 고정 배치한다.
- `주의`, `상담 필요`, `중복 확인` 같은 짧은 라벨(badge)을 활용한다.
- 긴 문단보다 1~3개의 핵심 bullet 형태를 우선한다.
- `caution_level` 값에 따라 색상 구분:
  - `info` → 회색/파란색
  - `warning` → 주황색
  - `consult` → 빨간색

### 쇼핑 섹션
- 전체 결과의 하단 보조 섹션으로 배치한다.
- 최저가 / 가성비 / 추천 판매처를 구분해서 표시한다.
- 가격 변동 가능성 안내 문구를 필수 표시한다: _"가격은 조회 시점 기준이며 실제 구매 시 다를 수 있습니다."_
- 구매 링크는 외부 링크로 연결한다.

## 3.3 필수 경고 문구

결과 화면 상단 또는 하단에 반드시 표시한다.

> "이 내용은 의료적 진단이 아닙니다."

> "복용 중인 약이나 임신·수유 중이라면 전문가 상담이 필요할 수 있어요."

# 4. UX 카피 원칙

## 온보딩 화면
- "10초 만에 추천받기"
- "딱 3가지만 알려주세요"
- "먼저 간단히 추천드릴게요"

## 보정 질문 화면
- "더 정확히 보려면 이것만 답해주세요"
- 진행 인디케이터로 "1/2", "2/2" 표시

## 결과 대기 화면
- 로딩 중 진행 상황을 단계별 텍스트로 안내한다.
  - "건강 목표를 분석하고 있어요"
  - "음식과 습관을 추천하고 있어요"
  - "영양 성분을 찾고 있어요"
  - "쇼핑 가격을 비교하고 있어요"

# 5. 프론트엔드 ↔ 에이전트 인터페이스 요약

| 방향 | 시점 | 데이터 형식 | 내용 |
|------|------|------------|------|
| FE → Agent | Level 1 입력 | JSON | 건강 목표, 나이대, 성별 |
| Agent → FE | 보정 질문 요청 (조건부) | JSON | 질문 목록 (최대 2개) |
| FE → Agent | Level 2 응답 (조건부) | JSON | 보정 질문 답변 |
| Agent → FE | 안전 질문 요청 (조건부) | JSON | 안전 확인 질문 목록 |
| FE → Agent | Level 3 응답 (조건부) | JSON | 안전 질문 답변 |
| Agent → FE | 최종 결과 반환 | JSON + Markdown | 추천 결과 + 쇼핑 정보 |

# 6. 에러 처리 원칙

| 상황 | 프론트엔드 처리 |
|------|---------------|
| 에이전트 응답 지연 | 로딩 화면 유지 + 타임아웃 안내 |
| 쇼핑 섹션 없음 | 쇼핑 섹션만 미표시, 나머지 결과는 정상 표시 |
| 영양 성분 추천 없음 | 해당 섹션 미표시, 음식·습관 섹션은 정상 표시 |
| 전체 실패 | 에러 메시지 표시 + 다시 시도 버튼 제공 |
