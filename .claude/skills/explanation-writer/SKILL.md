# 스킬: explanation-writer

## 역할
음식·생활습관·영양 추천 결과에 대한 자연어 설명을 생성한다.
주의사항 문구를 정리하고, 사용자가 이해하기 쉬운 건강 요약을 작성한다.

## 트리거 조건
최종 건강 추천 세트 확정 후 (Step 13)

## 처리 방식
에이전트 직접 판단 (LLM 생성)

## 입력
- `output/recommendation/final_health_plan.json`
- `output/risk/risk_flags.json`

## 출력
- `output/content/final_health_summary.md`

## 생성 원칙

1. **순서 고정**: 음식 → 생활습관 → 영양 성분 → 주의사항 순으로 작성
2. **과장 금지**: "반드시", "확실히", "치료된다" 같은 표현 사용 금지
3. **비의료적 표현 유지**: "도움이 될 수 있습니다", "고려해 볼 수 있습니다" 수준
4. **추천 이유 포함**: 각 항목마다 왜 이 추천이 나왔는지 1~2줄 설명
5. **주의사항 분리**: 추천 이유와 주의사항은 별도 섹션으로 분리
6. **위험 플래그 반영**: `consult_required: true`이면 상단에 상담 권고 문구 삽입

## 필수 포함 문구

결과 하단에 반드시 포함:
> "이 내용은 의료적 진단이 아닙니다."
> "복용 중인 약이나 임신·수유 중이라면 전문가 상담이 필요할 수 있어요."

## 출력 예시 구조

```markdown
# 나를 위한 건강 제안

## 추천 음식
...

## 생활습관 제안
...

## 영양 성분 제안
### [성분명]
**추천 이유**: ...
**주의사항**: ...

---
> 이 내용은 의료적 진단이 아닙니다.
```

## 성공 기준
- 추천 이유 누락 없음
- 금지 표현 없음
- 주의사항 누락 없음

## 실패 처리
- 자동 재시도 최대 2회
- 이후 에스컬레이션 (safety-reviewer 호출)
